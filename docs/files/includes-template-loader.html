template-loader.php

Template Loader

Handles template loading with a full override hierarchy:

  1. Child Theme: /themes/child/jpkcom-post-filter/{template}
  2. Parent Theme: /themes/parent/jpkcom-post-filter/{template}
  3. MU Plugin: /mu-plugins/jpkcom-post-filter-overrides/templates/{template}
  4. Plugin: /plugins/jpkcom-post-filter/debug-templates/{template} (when WP_DEBUG) /plugins/jpkcom-post-filter/templates/{template} (production)

jpkcom_postfilter_get_template_part(string $slug[, string $name = '' ][, array<string, mixed> $args = [] ]) : void

Similar to WordPress get_template_part() but uses the plugin's template hierarchy. Variables in $args are extracted into the template scope.

Usage examples: jpkcom_postfilter_get_template_part( 'partials/filter/filter-bar' ); jpkcom_postfilter_get_template_part( 'partials/list/list-cards', '', ['query' => $q] ); jpkcom_postfilter_get_template_part( 'partials/filter/filter-bar', 'sidebar', $args );

Parameters
$slug : string

Template slug (path without .php extension).

$name : string = ''

Optional template name/variation (appended as -{name}).

$args : array<string, mixed> = []

Variables to extract into the template scope.
